OAuth using In-Memory user id and password

Overview

  • The Resource Owner will ask the Client Application to get some data from the Resource Server.
  • The Resource Server asks the Resource Owner to authenticate itself and as for authorization to share data.
  • After successful authentication the Resource Server shares an authorization code with the client application
  • The authorization code is received as a request parameter and the ''resource server'' contacts the client application ''using the redirect uri''

    See files under com.isquarebsys.oauth

  • AuthorizationServerConfig.java: Creates Security Configuration
  • EmployeeClientControllerSecured.java: Provides DATA using the url->/oauth/apis/employees
  • EmployeeURLSecurityConfig.java: Secures the URL->/oauth/apis/employees
  • Access the Jersey Endpoint by calling /oauth/employees URL
